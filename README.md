# ملبّيك · أساس المصادقة والخصوصية

أساسٌ حقيقيٌّ (React + Supabase) يضمّ صفحة الهبوط، وتسجيل الدخول للأدوار الثلاثة (الإدارة / المشترك / العميل)، ونظام عزلٍ صارمٍ يجعل **كل عميلٍ يرى رحلات حملته فقط**.

---

## ١) التشغيل السريع

المشروع كاملٌ وجاهز (Vite + React). من داخل المجلد:

```bash
npm install
# انسخ .env.example إلى .env واملأ مفاتيح Supabase
npm run dev
```

> الاعتماديات معرّفةٌ في `package.json`، وملفات الخط مضمّنةٌ في `public/fonts/`.

## ٢) متغيّرات البيئة

أنشئ ملف `.env` في جذر المشروع:

```
VITE_SUPABASE_URL=https://YOUR-PROJECT.supabase.co
VITE_SUPABASE_ANON_KEY=YOUR-ANON-KEY
```

## ٣) قاعدة البيانات

افتح **Supabase ▸ SQL Editor** والصق محتوى `supabase/schema.sql` ثم شغّله. يُنشئ الجداول، الأدوار، التريغر، وكل سياسات الأمان (RLS).

## ٤) الخطوط

مضمّنةٌ مسبقًا في `public/fonts/` (١٢ وزنًا يطابق `malbeek-theme.css`) — لا حاجة لأي خطوةٍ إضافية.

## ٥) أثناء التطوير (مهم)

من **Supabase ▸ Authentication ▸ Providers ▸ Email** عطّل **Confirm email** مؤقتًا، حتى يُعيد التسجيل جلسةً فورية فيكتمل إنشاء الحملة/المعتمر تلقائيًا. (في الإنتاج فعّله وأضف خطوة إكمالٍ بعد التأكيد.)

## ٦) إنشاء حساب إدارة

سجّل مستخدمًا عاديًا، ثم رقِّه من SQL Editor:

```sql
update public.profiles set role = 'admin' where id = '<USER_UUID>';
```

## نموذج الخصوصية (جوهر الأمان)

| الدور | ما يراه |
|------|---------|
| **الإدارة** | كل المشتركين والرحلات |
| **المشترك** | رحلات ومعتمرو **حملته** فقط (حيث `subscriber_id` تابعٌ له) |
| **العميل** | رحلات **الحملة التي شاركت له الرابط** فقط |

العزل مفروضٌ على مستوى قاعدة البيانات عبر **RLS**، لا على الواجهة — فحتى لو حاول العميل الاستعلام مباشرةً، تُرجِع القاعدة رحلات حملته فقط. المفتاحان:

- `my_subscriber_id()` تُرجع حملة المستخدم الحالي من ملفّه الشخصي.
- سياسة `trips customer read scoped`: ‎`subscriber_id = my_subscriber_id()` — هذا السطر هو ضمان «كل عميلٍ يرى حملته فقط».

عند فتح العميل للرابط `/j/:slug`، يُحلّ الـ slug إلى الحملة، ويُنشأ حسابه مربوطًا بها عبر بيانات التسجيل، فيتولّى التريغر ضبط `subscriber_id` في ملفّه — ومنها يَسري العزل تلقائيًا.

---

## تدفّق الاستخدام

1. **الزائر** يفتح صفحة الهبوط ← «ابدأ التجربة» أو «تسجيل الدخول».
2. **المشترك** يسجّل (تجريبي ١٤ يوم) ← تُنشأ حملته ← لوحته فيها **رابط مشاركة** ينسخه ويرسله لعملائه.
3. **العميل** يفتح رابط المشترك ← يعبّئ بياناته مرةً ← يدخل لوحته في أي وقتٍ لاحقًا ويرى رحلات تلك الحملة فقط.
4. **الإدارة** تدخل بحسابها للإشراف على المنصّة.

## خريطة الملفات

```
malbeek-foundation/
├─ supabase/
│  └─ schema.sql              ← الجداول + RLS + التريغر + العرض العام
└─ src/
   ├─ main.jsx                ← نقطة الدخول (Router + AuthProvider)
   ├─ lib/
   │  └─ supabaseClient.js
   ├─ app/
   │  ├─ AuthProvider.jsx     ← الجلسة + الدور + الحملة
   │  ├─ RequireAuth.jsx      ← حارس المسارات حسب الدور
   │  └─ App.jsx              ← تعريف المسارات
   ├─ components/
   │  └─ CompassMark.jsx      ← شعار البوصلة الثمانية
   ├─ styles/
   │  ├─ malbeek-theme.css    ← خط ثمانية + ألوان ملبّيك
   │  └─ app.css              ← أنماط الشاشات واللوحات
   └─ pages/
      ├─ auth/
      │  ├─ AuthShell.jsx     ← هيكل شاشة المصادقة الفخم
      │  ├─ Login.jsx         ← دخول كل الأدوار (توجيهٌ حسب الدور)
      │  ├─ Signup.jsx        ← تسجيل مشترك (تجريبي)
      │  └─ CustomerJoin.jsx  ← انضمام العميل عبر /j/:slug
      └─ app/
         └─ Homes.jsx         ← لوحات: AdminHome / SubscriberHome / CustomerHome
```

> صفحة الهبوط التسويقية مستقلّة في `malbeek-landing.html` — اخدمها على الجذر `/`، أو انقلها لاحقًا إلى مكوّن React.
