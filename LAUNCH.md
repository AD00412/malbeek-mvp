# دليل الإطلاق التجريبيّ · ملبّيك

قائمةٌ عمليّةٌ لتهيئة المنصّة للتجربة الأولى مع مشتركين حقيقيّين، ثمّ خطوات
ما بعد التجربة (الدومين، البريد، الدفع، المحفظة) عند الإطلاق الرسميّ.

---

## أ) الإعداد للتجربة (مرّةً واحدة)

1. **مشروع Supabase**: أنشئ مشروعًا على https://supabase.com — مجّانيٌّ يكفي للتجربة.
2. **المتغيّرات** (`.env` في جذر المشروع — من `Project Settings ▸ API`):
   ```
   VITE_SUPABASE_URL=https://YOUR-REF.supabase.co
   VITE_SUPABASE_ANON_KEY=YOUR-ANON-KEY
   ```
   > العنوان الأساسيّ فقط (بلا `/rest/v1/`)، وبلا علامات اقتباس. لا تستخدم `service_role` في الواجهة أبدًا.
3. **قاعدة البيانات**: `SQL Editor` ▸ الصق كامل `supabase/schema.sql` ▸ Run.
   آمنٌ لإعادة التشغيل بعد أيّ تحديث.
4. **المصادقة**: `Authentication ▸ Providers ▸ Email` مفعَّل، و**Confirm email معطَّل أثناء التجربة**
   (ليدخل المستخدم فورًا بلا تأكيد بريد). فعّله في الإطلاق الرسميّ.
5. **حساب الإدارة** (لك): سجّل حسابًا عاديًّا، ثمّ في SQL Editor:
   ```sql
   update public.profiles set role = 'admin' where id = '<USER_UUID>';
   ```
6. **البناء والنشر**: `npm install` ثمّ `npm run build` ▸ انشر مجلّد `dist/`
   على أيّ استضافةٍ ثابتة (Vercel/Netlify/Cloudflare Pages). اضبط متغيّرات البيئة في لوحة الاستضافة.
   > للتطبيقات أحاديّة الصفحة (SPA): أعد توجيه كلّ المسارات إلى `index.html`.

## ب) خطّة الاختبار السريعة (الأدوار الثلاثة × الأجهزة)

اختبر على **جوّال + لوح + حاسوب**:

- **المشترك**: تسجيل ← إنشاء رحلة ← إضافة معتمر/استيراد ← خريطة المقاعد ← تعدّد باصات ←
  فنادق وتسكين ← كشف PDF/Word ← مسح صعود ← استرداد ← دعوة عضو فريق ← التقرير الماليّ.
- **المعتمر**: فتح رابط `/j/:slug` ← تسجيل ← اختيار مقعد ← دفع (متجر + إرفاق إيصال) ←
  تذكرة (QR/مشاركة/حفظ/تقويم) ← تتبّع الحالة ← إلغاء/استرداد.
- **عضو الفريق**: فتح رابط `/join-team/:id` ← قبول ← دخول لوحة الحملة.
- **الإدارة**: دخول ← نظرة عامّة على الحملات ← صندوق التغذية الراجعة.

تحقّق من: لا شاشات بيضاء، لا تعليق، رسائل خطأ عربيّة واضحة، أزرار ملمسها ≥ ٤٤px،
لا تمرير أفقيّ على الجوال، RTL سليم.

## ج) مشاركة المنصّة مع المشتركين

- كلٌّ يسجّل من صفحة الدخول/التسجيل ← تُنشأ حملته تلقائيًّا.
- يشارك رابط الحجز `/(j)/slug-حملته` مع معتمريه.
- لإضافة موظّفٍ لفريقه: «الفريق والصلاحيّات» ← دعوة بالبريد ← نسخ الرابط وإرساله ← يقبله الموظّف.

---

## د) ما بعد التجربة (الإطلاق الرسميّ)

| الخطوة | ما تحتاجه |
|--------|-----------|
| **الدومين** | اشترِ دومينًا واربطه بالاستضافة + حدّث Supabase ▸ Authentication ▸ URL Configuration (Site URL + Redirects). |
| **بريد الدعوات** | إعداد SMTP مخصّصٍ على الدومين في Supabase ▸ Auth ▸ SMTP (أو Edge Function مع مزوّدٍ مثل Resend). |
| **تأكيد البريد** | فعّل Confirm email، وأضف خطوة إكمالٍ بعد التأكيد. |
| **تذكيرٌ تلقائيّ قبل الرحلة** | فعّل pg_cron (Database ▸ Extensions) ثمّ أعد تشغيل `schema.sql`. |
| **بوّابة دفعٍ مؤكَّدة** | انشر `supabase/functions/create-payment` + `payment-webhook` بمفاتيح بوّابتك (Moyasar/Tap). |
| **محفظة آبل/جوجل** | شهاداتُ توقيعٍ + حساباتٌ تطويريّة. |
| **أمان الإطلاق** | راجع/دوّر المفاتيح، تأكّد أنّ Confirm email مفعّل، وأنّ `.env` غير منشور. |

---

## هـ) ملاحظاتٌ أمنيّة (مؤكَّدة)
- العزل بين الحملات مفروضٌ على مستوى القاعدة (RLS) — لا على الواجهة.
- الأدمن يُمنح يدويًّا من القاعدة فقط؛ لا تصعيد امتيازٍ عبر التسجيل.
- الواجهة تستخدم مفتاح `anon` فقط؛ لا أسرار في الكود.
- المنصّة لا تتطلّب بوّابة دفعٍ لتعمل — الدفع عبر المتجر + إثبات الإيصال يكفي للتجربة.
